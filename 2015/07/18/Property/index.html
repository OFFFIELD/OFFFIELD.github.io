@Property

修饰符用法以及常见问题

修饰符

  • readwrite

    由系统创建setter&getter方法，默认值为readwrite，即property属性默认创建setter&getter方法

    @property (readwrite) NSObject *mObj;

  • readonly

    由系统自动创建getter方法，不创建setter方法

    @property (readonly) NSObject *mObj;

  • @synthesize

    Xcode4.5之后可以不写

    //在类成员变量中查找mObj和_mObj,若都没有，自动生成私有变量_mObj;
@synthesize mObj;

//如果想重新定义变量名
@synthesize  mObj = _mObjPrivate;

  • setter & getter

    重写setter或getter方法，覆盖readwrite或readonly所创建对应的方法

    //ViewController.h

#import <UIKit/UIKit.h>

@interface ViewController : UIViewController

@property (nonatomic, strong, getter = mGetNum, setter = mSetNum:) NSNumber *num;

@end

//ViewController.m

@interface ViewController ()
{
   //重新定义了setter和getter，@synthesize就会失效，需要重新定义私有变量
   NSNumber *_num;
}

@end

@implementation ViewController

- (void)viewDidLoad
{
  [super viewDidLoad];

  //只有用点语法才会调用setter方法
  self.num = @2;

  NSLog(@"%@",self.num);
}

//setter和getter里不可以用点语法，会导致死循环
- (void)mSetNum:(NSNumber *)num
{
      _num = num;
}

- (NSNumber *)mGetNum
{
   return _num;
}

@end

  • assign

    修饰基础类型：id,NSInteger,CGFloat,int,long,float,double…

  • weak

    弱引用，当所指向对象没有其他强引用，weak所修饰的指针自动置为nil，避免循环引用

  • strong(ARC),retain(MRC)

    指针拷贝，引用计数+1，可用于NSObject所有子类

  • copy

    可用于实现了NSCoying,NSMutablecopying的对象，内容拷贝，原对象引用计数不变，创建新对象，与原对象互不干扰

    iOS Developer Library:Copying Collections

@interface Man : NSObject
@property (strong, nonatomic) NSString *nameStrong;
@property (copy, nonatomic) NSString *nameCopy;
@end

- (void)test
{
  NSMutableString *manName = [NSMutableString stringWithString:@"Tom"];

  Man *man = [[Man alloc] init];

   man.nameStrong = manName;

   man.nameCopy = manName;

  [manName setString:@"Jack"];

   //输出结果为Jack
   NSLog(@"man name %@", man.nameStrong);

   //输出结果为Tom
   NSLog(@"man name %@", man.nameCopy);
}

  • atomic

    提供多线程安全,防止多线程操作数据出错，防止在写入未完成的时候被另外一个线程读取，造成数据错误

  • nonatomic

    如果不需要多线程操作，采用此修饰符可以提高性能
